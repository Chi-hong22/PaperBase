## 🔍 问题修复总结

### P0 级修复（3/3 完成）

#### 1. query topic 覆盖率不足 ✅
**问题**：只能查询到 62.5% 的论文（5/8）

**根因**：正则表达式只匹配 `p_[0-9a-f]{12}$` 格式，遗漏自定义后缀节点

**修复方案**：
```python
# 修改前
r'^p_[0-9a-f]{12}$'

# 修复后
r'^p_[0-9a-f]{12}(_paper)?$'  # 兼容新旧格式
```

**验证结果**：覆盖率提升至 100% (9/9)

---

#### 2. query topic 结果重复 ✅
**问题**：可能出现重复的 paper_id

**修复方案**：在 `query.py:241-244` 添加去重逻辑

**验证结果**：所有查询无重复，paper_id 唯一性约束生效

---

#### 3. 新摄入论文未自动索引 ✅
**问题**：摄入后需手动运行 `paperbase index`

**修复方案**：ingest 命令自动触发索引更新

**验证结果**：
- ✅ 自动生成 chunks.jsonl
- ✅ 自动更新 FTS5 索引
- ✅ 自动更新知识图谱

---

### P1 级修复（1/2 完成）

#### 4. query related 返回空结果 ⚠️
**问题**：42.9% 论文无关联，图谱更新后仍为 33.3%

**根因**：**Graphify 设计限制**
- 只提取论文内概念关系

- 需要引用解析器支持

## 🐛 发现并修复的 Bug

### Bug: authors 字段处理错误

**位置**：`src/paperbase/cli/commands/query.py`

**现象**：query topic 显示 authors 时报错

**根因**：
- Registry 存储 JSON 字符串
- 未反序列化直接操作
- 导致类型错误

**修复**：
```python
# 统一处理 str/list/dict 类型
if isinstance(authors, str):
    authors = json.loads(authors)
if isinstance(authors, list):
    authors = ", ".join([a.get('name', a) if isinstance(a, dict) else a for a in authors])
```

**状态**：✅ 已修复
